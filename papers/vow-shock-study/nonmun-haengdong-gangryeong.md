# 논문 행동 강령

지칭 규칙
논문 내에서는 "본 연구", "본 논문" 대신 반드시 **"이 연구", "이 논문"**으로 통일할 것

참고문헌 표기
참고문헌 번호는 본문 등장 순서에 따라 번호 부여
본문 내 표기는 [1], [2] 등 괄호 포함 숫자 표기법을 사용할 것

각주 처리
초안 작성 시에는 출처 표기를 위해 각주 사용 가능
그러나 최종 논문에서는 각주는 전부 제거하고 참고문헌만 유지할 것

문단 구성
모든 장(Ⅰ, Ⅱ 등)은 문장 단위의 설명 서론으로 시작하고, 중간에 표, 그림, 코드, 수식을 배치하되
이들을 단락 설명과 함께 반드시 해석해야 한다

줄바꿈 규칙
문장마다 줄바꿈하지 않는다(Enter 금지). 문단이 끝날 때만 줄바꿈하고, 문단 사이에는 빈 줄 1줄을 둔다.

그림 및 표 규칙
모든 그림과 표는 그림 1., 표 1.과 같은 한글 캡션 + 영어 캡션 병기 형식으로 작성할 것
표는 최대한 줄글이 아닌 항목 중심, 수치 중심으로 간결하게 정리할 것

코드 및 수식
코드 설명 시 함수는 최소 1~2줄 예시와 함께 작성할 것
수식을 제시하는 경우에는 그 아래 반드시 의미를 자연어로 해설할 것

문장 표현
“제안된”, “확인되었다”, “보여준다” 등 객관적 중립 표현을 사용할 것
확신이 부족하거나 근거가 없는 문장은 지양 (“유의미할 수 있다” → “확인되지 않았다”로 수정)

단락 길이 제한
하나의 문단은 5줄 이하로 유지할 것 (가독성과 편집을 위한 기준)

금지 표현(용어)
- 1인칭 표현 금지: “우리는”, “우리의” 등 → 대체: “이 연구는 … 다룬다/분석한다/제안한다”.
- 조직 지칭 금지: “연구팀”, “저자들은” 등 → 대체: “시스템/설계/프레임워크는 …”.
- 용어 통일: “본 연구/본 논문” → “이 연구/이 논문”.
- 서론에서 개별 논문명·저자 직접 언급 금지(동향 요약 + 인용 번호만 사용).
- 인용 위치: 문장 말미 ‘마침표 직전’에 배치(예: …관찰되었다[1].), 문단 단위 배치 금지.

관련 연구(2장) 예외 규정
- 관련 연구 섹션에서는 개별 논문 제목·저자 직접 언급을 허용한다.
- 다만 숫자 인용([1] 등) 표기는 생략하고, 서론·참고문헌과의 매칭만 보장한다.
- 각 논문 요약은 “핵심 아이디어 → 강점 → 한계(본 연구와의 차별점)” 순서로 2–3문장 이내로 작성한다.
