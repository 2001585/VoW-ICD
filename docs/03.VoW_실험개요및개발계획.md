 Ⅲ. 실험 개요 및 개발 계획
본 섹션은 CLI 기반의 정량 실험 프레임워크를 구축하기 위한 실험 구조 개요와 스펙 주도 개발 절차를 정리한다.
 3.1 전체 실험 구조 개요
본 실험은 LM Studio에서 구동되는 다수의 로컬 LLM을 CLI 기반으로 제어하며, 다중 에이전트가 자율적으로 협력, 기
여, 이탈, 회복 등의 사회적 행동을 자연어로 수행하는 시뮬레이션을 구성한다. 에이전트는 발화-행동-기억 구조를 따르
며, 구조화된 JSON 출력으로 정량적 분석이 가능하도록 한다.
 3.2 Codex 기반 스펙 주도 개발 절차 (인터넷 기준 최신 방식 반영)
 1단계: AGENTS.md 생성
AGENTS.md 파일을 프로젝트 루트에 생성하여 다음을 포함한다: - 프로젝트 목적 - 주요 컴포넌트 정의 - 개발 방식: 스
펙 → 체크리스트 → 코드 작성 → 반복 방식 고수 - 매 Phase 종료 시 체크리스트 반드시 갱신
2단계: docs/ 폴더 생성 및 스펙 명세 작성
docs/ 디렉토리를 생성하고 각 구성요소별 스펙 문서 (*.md)를 작성한다. 예시: - 
→ 에이전트 분리 구조, 상태 저장 방식 - 
docs/AgentManager.md
 docs/LLMWrapper.md → LM Studio API 호출 구조, 포트별 요청 분리 
docs/TurnManager.md → 라운드 구성, 대화 흐름 - 
docs/Evaluator.md → 정량 지표 계산 공식, 로그 필
드 명세 - 
docs/CLI.md → CLI 사용자 인터페이스 동작 흐름
이 방식은 2025년 기준 Codex 기반 프로젝트의 스펙 선도방식으로 HuggingFace, LangChain 문서화 표준에서 채택
된 접근이다 (참조: huggingface.co/docs, langchain.com/specs)
